可解决：

痛点2：长指令目标遗忘、走回头路

痛点3：无学习能力、规划稳定性差

解决：

1. 框架嵌入点：
    
    ```nasm
    【输入】用户导航指令
        ↓
    【MLLM】摄像头2D图→场景文本描述（完全不动，和之前一模一样）
        ↓
    【核心优化对象：LLM决策层】（RL只优化这里）
        输入：用户指令+当前场景描述+历史路径
        输出：高层规划动作
        ←【RL优化模块】：用PPO/GRPO算法，基于导航结果的奖励信号，微调LLM的规划策略
        ↓
    【底层控制】动作转电机指令→仿真/真实环境执行（完全不动，和之前一模一样）
        ↓
    【闭环反馈】执行结果→新的画面→MLLM转文本→喂给LLM，同时把「这一步动作的好坏」传给RL模块做奖惩
    ```
    
2. 奖励函数设计

| 奖惩类型 | 触发条件 | 奖励 / 惩罚数值 | 解决的具体问题 |
| --- | --- | --- | --- |
| 【最终正奖励】 | 机器人成功到达目标终点，SR 达标 | +100 | 让 LLM 学会 “最终要走到目标”，从根源上避免遗忘目标 |
| 【路径效率正奖励】 | 每走一步，离目标的直线距离比上一步更近 | +5 / 步 | 让 LLM 学会往目标方向走，避免瞎绕路、走回头路 |
| 【回头路惩罚】 | 走了之前已经走过的节点 / 位置 | -10 / 步 | 直接惩罚走回头路的行为，LLM 很快就会学会不重复走老路 |
| 【无效动作惩罚】 | 连续 3 步原地打转、无意义左转 / 右转 | -20 | 惩罚瞎规划的无效动作，让 LLM 的决策更稳定 |
| 【指令偏离惩罚】 | 走的路径和指令要求的关键节点不符（比如指令要求 “经过玄关”，结果没走玄关） | -50 | 让 LLM 记住指令的核心要求，避免长指令遗忘关键目标 |
| 【失败惩罚】 | 导航超时、撞墙、走进死胡同 | -80 | 惩罚完全错误的规划，让 LLM 学会避开无效路径 |
| 【子目标完成正奖励】 | 每成功完成一个指令拆解的子目标 | +30 | 让 LLM 学会盯着子目标走 |
| 【子目标拆分惩罚】 | 拆的子目标无法完成、不符合环境逻辑 | -40 | 让 LLM 学会更合理的拆分方式 |

**<方案3> 实时反馈的RL纠错（之后）**

纯 Prompt 的 LLM，只有走完一步、发现完全错了，才会回头重规划，而这个方案用 RL 训练 LLM 学会

**实时纠错**：

每走一步，就用当前环境的反馈，判断当前规划对不对，不对就立刻修正，不用等走错了再回头。简单说：之前的 LLM 是 “先规划、再执行、错了再改”，这个方案是 “边执行、边判断、边优化”，完全贴合真实机器人的导航场景。

**核心实现方式：**

1. 给 LLM 的输入里，加一个「当前规划置信度」的输出，让 LLM 每一步都判断 “当前的规划能不能完成目标”；
2. 用 RL 优化这个置信度判断和纠错策略，用「纠错后的路径效率、最终成功率」做奖励信号；
3. 让 LLM 在仿真里试错，学会 “什么时候该修正规划、该怎么修正”，而不是靠 Prompt 里固定的 “每 3 步核对一次”。

RL算法选择：GPPO
