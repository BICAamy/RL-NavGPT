## **1. RL成员**

| 标准 RL 术语 | 在我们的 VLN 方案里是什么 | 具体解释 |
| --- | --- | --- |
| **智能体（Agent）** | 带 LoRA 适配器的 LLM 决策层 | 核心优化对象，只有 LoRA 的参数会被更新，LLM 主干权重完全冻结，和原版 NavGPT 的 LLM 是同一个模型，只是加了个可训练的小插件。 |
| **环境（Environment）** | 黑盒打包的「VLN 仿真环境 + 固定 MLLM + 固定 Prompt 管理器」 | 和原版 NavGPT 一模一样，负责：
1. 给智能体返回当前的「状态」（打包好的 Prompt）；
2. 执行智能体输出的「动作」；
3. 给智能体返回「奖励」和「下一状态」。 |
| **状态（State, st）** | 原版 NavGPT 的完整输入 Prompt | 就是之前看到的：
「系统规则 + 用户导航指令 + MLLM 转的当前场景文本描述 + 历史路径总结」，打包成一个字符串，直接喂给 LLM。 |
| **动作（Action, at）** | LLM 输出的高层规划动作 | 和原版 NavGPT 完全一致：比如输出 `action_maker("viewpoint_id_123")` （选下一个导航点），或者 `Finish!` （停下）。 |
| **奖励（Reward, rt）** | 我们之前设计的 VLN 专用奖励函数 | 每执行一步动作，环境就根据
「导航进度、是否走回头路、是否偏离指令」返回一个数值奖励，比如：- 离目标更近：+5- 走了回头路：-10- 成功到达终点：+100- 超时 / 撞墙失败：-80 |
| **轨迹（Trajectory）** | 一次完整导航的 (s0,a0,r0,s1,a1,r1,...,sT,aT,rT) | 智能体在仿真里从起点跑到终点（或超时失败）的完整过程，存下每一步的状态、动作、奖励，用来做 RL 训练。 |

## 2. 训练流程

### 阶段 2：在线交互收集数据

这个阶段就是让**带 LoRA 的 LLM 智能体**，在**固定的黑盒环境**里，像玩游戏一样反复跑导航，收集大量的「轨迹数据」，用来训练：

1. **重置环境**：从 R2R 训练集里随机选一条导航指令，把机器人放到起点，环境返回初始状态 s0（初始的 Prompt）；（R2R提供的是「场景、指令、起点、终点」）
2. **智能体决策**：把 s0 喂给带 LoRA 的 LLM，LLM 输出动作 a0；
3. **环境执行动作**：仿真环境执行 a0，机器人移动到下一个位置，MLLM 把新画面转成文本，Prompt 管理器打包成新的状态 s1；
4. **环境返回奖励**：根据机器人的移动情况，用我们的奖励函数计算奖励 r0（比如离目标更近了 + 5，走回头路 - 10）；
5. **存下轨迹数据**：把 (s0,a0,r0,s1) 存到经验回放池（或直接存成轨迹文件）；
6. **循环交互**：重复步骤 2-5，直到机器人到达终点（成功）或超时（失败），完成**一条完整的轨迹**；
7. **批量收集**：重复步骤 1-6，收集成百上千条轨迹数据，攒够一个 Batch 的训练数据。

---

### 阶段 3：RL 训练更新参数（核心，只更新 LoRA）

这个阶段就是用收集到的轨迹数据，用 GRPO/PPO 算法**只更新 LoRA 适配器的参数**，LLM 主干权重完全不动：

1. **加载轨迹数据**：从经验回放池里取出一个 Batch 的轨迹数据；
2. **计算回报（Return）**：用奖励函数计算每一步的「累积回报」（也就是这一步动作对最终成功的贡献）；
3. **GRPO/PPO 算法更新**：用 Hugging Face 的 TRL 库里的 `GRPOTrainer`，把轨迹数据喂进去：
    - 策略网络：带 LoRA 的 LLM；
    - 价值网络：可以用同一个 LLM 加一个小的价值头（TRL 库自动搞定）；
    - 损失函数：GRPO/PPO 的标准损失函数，专门优化 LLM 的策略，让它输出能拿到更高奖励的动作；
4. **只更新 LoRA**：训练过程中，LLM 的主干权重 100% 冻结，只有 LoRA 适配器的参数会被梯度下降更新；
5. **验证与保存**：每训练几个 Epoch，就在 R2R 的验证集上测一下 SR/SPL 指标，保存指标最好的 LoRA 权重；
6. **循环迭代**：重复「阶段 2 收集数据→阶段 3 训练更新」这个过程，直到验证集指标不再上涨，训练完成。

---

不用自己写 RL 的底层代码，直接用 Hugging Face 的 **TRL（Transformer Reinforcement Learning）库**，里面已经封装好了 `GRPOTrainer`，专门用来做 LLM 的 RL 微调，你只需要：

1. 写一个简单的「环境交互函数」，负责让 LLM 在仿真里跑导航、收集轨迹；
2. 写好我们之前设计的奖励函数；
3. 调用 `GRPOTrainer`，把你的 LLM、LoRA 配置、训练数据传进去，一键开始训练。

---

## 3. 加入 RL 后的「推理流程」

**核心重点：推理的时候，RL 模块完全不工作！** 

训练好之后，只需要用「训练好的带 LoRA 的 LLM」替换掉原版 NavGPT 里的 LLM，其他所有环节（MLLM、Prompt 管理器、仿真环境）完全不动，推理流程和原版 NavGPT100% 一致，就是纯前向推理，没有任何 RL 的东西。

## 完整推理流程（4 步，和原版 NavGPT 完全对齐）

1. **加载模型**：加载「冻结的 LLM 主干 + 训练好的 LoRA 适配器」，合并成一个推理用的 LLM（或者直接用 PEFT 库加载 LoRA，不用合并）；
2. **环境初始化**：加载一条新的导航指令，把机器人放到起点，MLLM 把初始画面转成文本，Prompt 管理器打包成初始 Prompt；
3. **纯前向推理循环**：
    - 把当前 Prompt 喂给训练好的 LLM；
    - LLM 纯前向推理，输出「Thought+Action」（和原版 NavGPT 一模一样）；
    - 执行动作，机器人移动，获取新画面，MLLM 转文本，打包新 Prompt；
    - 重复这个过程，直到 LLM 输出 `Finish!`；
4. **结束**：导航完成，和原版 NavGPT 的区别只是 LLM 的规划更稳定、成功率更高。

## 4. 完整流程

1. **环境准备**：加载 R2R 训练集，把 3D 场景导入 Habitat 仿真环境，拿到训练集里的几千条导航指令（任务题库）；
2. **模型初始化**：加载 NavGPT 用的开源 LLM（比如 Qwen2.5-7B-Instruct），给它插上 LoRA 适配器，**100% 冻结 LLM 主干权重，只放开 LoRA 的参数可训练**；
- **RL 交互收集数据**：从 R2R 训练集里随机抽一条导航指令，把机器人放到对应起点，让带 LoRA 的 LLM 在仿真里跑导航，和环境交互，收集完整的轨迹数据（每一步的状态、动作、奖励、下一状态）；
- **RL 训练更新参数**：用收集到的轨迹数据，调用 TRL 库的 GRPO 算法，**只更新 LoRA 适配器的参数**，LLM 主干完全不动；
- **循环迭代**：重复「抽指令→交互收数据→训 LoRA」的过程，每训几轮，就在 R2R 验证集上测 SR/SPL 指标，保存效果最好的 LoRA 权重；
- **推理验证**：训练完成后，把训好的 LoRA 插回原来的 LLM，在 R2R 测试集上跑推理，和原版 NavGPT 对比指标，验证你的优化效果。
